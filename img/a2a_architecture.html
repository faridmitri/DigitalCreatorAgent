Digital Worker Agent — A2A Architecture

An Orchestrator LlmAgent coordinates three independent specialist agents over the Agent-to-Agent (A2A) protocol. Each specialist is its own Cloud Run Service publishing an Agent Card. The orchestrator delegates through a single send_message tool so it keeps control across all three calls.

Open source — CI/CD ships for both architectures, you choose the mode. Two GitHub Actions workflows live in the repo: deploy-a2a.yml (this diagram) and deploy-monolith.yml (monolith diagram). The A2A workflow runs manually by default; its push: trigger is included but commented out — uncomment it to also auto-deploy A2A on push to a2a/**. The monolith auto-deploys on push out of the box.
